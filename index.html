Statement

My thesis asks how a cinematic scene, staged inside a period setting, carries a game's story without a line of dialogue. My practice asks the same question of a product: what does this screen need to say, and in what order, for someone to trust it enough to keep going? I answer that question by directing AI tools through full builds — briefing the logic, the data model, and the interface the way I'd brief a small crew — and shipping the result as a real, working app rather than a mockup.

EXT. STREET GAME — FOUR ON FOUR

Uzun Eşek Online

A Turkish street game, turned into a 4v4 turn-based mobile match

Uzun Eşek is a schoolyard game played by two teams of four — one side forms a human chain to vault over, the other tries to break it by landing wrong. This project rebuilds that contest as a 4v4 turn-based mobile match: card-based team plays decide each round, a stability calculation stands in for the chain's balance, and Unity's Netcode for GameObjects carries it online. The rules engine is pure C#, built and tested independent of the Editor — 2,000 simulated matches surfaced a first-pass balance skew that a rebalancing pass brought to roughly 52/48, not yet confirmed with real players.

INT. LIVING ROOM — FOUR BUTTONS, NO MENU

Kolayfon

A phone experience stripped down to four buttons, built native and in parallel for elderly users

Same brief, two platforms, built the way each one actually works: on Android, Kolayfon replaces the home screen outright — a real launcher with an intent-filter for HOME/DEFAULT — because Android allows that. iOS doesn't, so the iOS build is a companion app plus a WidgetKit home-screen widget instead. Both sides land on the same four actions — Call, Message, Camera, Emergency — with a confirmation screen before every call so a shaking hand can't misdial, and a real ACTION_CALL behind the emergency button on Android.

INT. AUDIT DASHBOARD — SIGNAL FROM NOISE

SEO Auditor

Point it at a URL, get back what actually needs fixing — not a wall of metrics

A small FastAPI + Next.js tool that fetches a page, extracts its SEO/content signals — title and meta-description length, heading structure, image alt-text coverage — and hands them to Claude to turn into a prioritized, plain-language action list instead of a raw metrics dump. Runs bring-your-own-key: each user's Claude API key lives only in their browser's local storage and is never stored or logged server-side.

How a Take Gets Made

  1. Brief

    Define the person's problem and the smallest version of a v1 that actually solves it — before any screen exists.

  2. Direct

    Describe the screens, logic, and data model to an AI build partner in the language of product, not syntax.

  3. Build

    Flutter/Firebase or Unity scaffolding assembled and wired end-to-end the same day, not mocked.

  4. Cut

    Test on device, cut what doesn't serve the user, ship the next take.
